設定不要です。接続ボタンを押すとすぐに使えます。

GitHub Gist に各ページを JSON ファイルとして保存します。
完全無料・プロジェクト数制限なし。60秒ごとに他の端末と同期。
  1. gist.github.com を開き「＋」で新しい Gist を作成
    （ファイル名: wiki-init.md、内容: # My Wiki、Secret で OK）
  2. 作成後の URL から Gist ID をコピー
    例: https://gist.github.com/yourname/abc123def456
  3. Personal Access Token を作成
    スコープ: gist のみチェック → Generate token → コピー
トークンはブラウザのローカルストレージに保存されます
Firebase Firestore を使ったリアルタイム同期。完全無料（1GB / 50K読み取り/日）。
プロジェクト数の制限もなく Supabase の代替として最適です。
  1. console.firebase.google.com でプロジェクト作成
  2. 左メニュー「構築」→「Firestore Database」→「データベースの作成」
    モード: テストモード で開始（後で変更可）
  3. プロジェクトの設定（歯車アイコン）→「全般」タブ →
    「マイアプリ」セクションの「ウェブアプリを追加」→ config をコピー
config オブジェクト内の apiKey と projectId の値だけ入力すれば OK
⚠️ Supabase 無料プランはプロジェクト数が2つまでです。 プロジェクトに空きがある場合はご利用いただけます。
  1. supabase.com でプロジェクトを作成
  2. SQL エディタで下の SQL を実行
  3. Project Settings → API → URL と anon key をコピー
CREATE TABLE wiki_pages ( id uuid DEFAULT gen_random_uuid() PRIMARY KEY, path text UNIQUE NOT NULL, title text NOT NULL DEFAULT '', content text DEFAULT '', created_at timestamptz DEFAULT now(), updated_at timestamptz DEFAULT now() ); ALTER TABLE wiki_pages ENABLE ROW LEVEL SECURITY; CREATE POLICY "public_all" ON wiki_pages FOR ALL USING (true) WITH CHECK (true); ALTER PUBLICATION supabase_realtime ADD TABLE wiki_pages;
